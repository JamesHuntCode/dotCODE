Introduction to JavaScript:

Why learn JavaScipt?

Good question, why should you learn JavaScript in 2017? JavaScript is an amazing programming language to learn in 2017 because it is the language currently second in demand from employers. Learning JavaScript will increase your job prospects and give you a higher chance at getting that job. JavaScript is the future. There are so many libraries for JavaScipt which makes the language, well... usable for almost anything. Learning vanilla JavaScript with us here is a fantastic idea to get you started on a whole journey of programming.

What is JavaScript used for?

Looking at where we are now, JavaScript can be seen everywhere. You see that cool animation on that webpage that you like? Yeah, that was JavaScript. Do you like how fast you can log into many different websites without having to wait for years? Well, that was most likely Node.js (a popular server-side JavaScript framework) that allowed you to do that. JavaScript can now be used client and server-side to create a smooth, dynamic and modern webpage. Get started soon because JavaScipt is only going to become more and more popular.

Is JavaScipt hard to learn?

As far as programming languages go, JavaScript is fairly high-level, meaning that it really isn't that hard to learn. If you imagine low-level programming to be machine code (1's and 0's), JavaScript is extremely high level, meaning that it is closer to how we interpret the code than the computer. It definately isn't the easiest language to learn, but it certainly is very easy to learn if you have the right mindset. This course will walk you through everything from defining variables to working with objects so make sure you pay attention and most importantly... TAKE NOTES.

JavaScript Comments:

In the image shown above you will see a few different examples of comments in JavaScript.

Comments in JavaScript are no different to comments in other programming languages.

Comments are used for exaplaining certain parts of code and for helping us understand what we are doing.

For example (when we move onto functions, comments are far more useful), we will use comments to remind us what a specific function will do.

JavaScript Data Types:

JavaScript has 6 official data types that we as developers can use.

In the image above I have shown you all of the different data types in JavaScript and what they look like.

Here is a breakdown of what they are called and what kind of data they store:

STRING: A string is anything that is enclosed within a set of speech marks. This can include numbers.

NUMBER: A number value is just as you may have guessed, any numberic value assigned to a variable.

BOOLEAN: A boolean value, or a "bool", is a value that is either true or false.

OBJECT: We will learn about objects in future lessons, but it is useful to know they reserve their own data type.

UNDEFINED: Undefined becomes present when a variable has been declared, but has not been assigned any data to hold.

NULL: In JavaScript (other languages very), null is linked to no identification at all, it often points to where data should be, however is not.

The good thing about JavaScript is that it can tell what data it is storing and will assign it a data type for you, you will never have to define the data type for any variable in JavaScript.

JavaScript If/Else Statements:

JavaScript Code:

The if and else statement is something that every single programmer needs to learn. Above you will see a very useful example of how it works.

It works similar to basic English, if this happens, do this. If that happens, then do that instead.

Inbetween the brackets that come after you write if or else if you will need to write your condition. That condition can be anything you want.

You can write an unlimited amount of else if statements, however you can only have one if statement and one else statement.

When writing your result, (the part that goes in the squigly brackets), you can put anything you want, but for an example, use console.log so you can see what your code is doing.

To run you through what happens when we run an if/else if/else statement in JavaScript, this is how it is broken down:

The variable is checked against each condition within the if/else if statements.

The first condition that equates to true will end the analysis and the code for that condition will be run.

If no condition is met at all, the code that is in the else brackets will be run.

Multiple conditions may be met, however, the first one to match is the one that will produce its result.

While loops are one of the easier styles of looping because they are easy to understand, if you think of it like; while this is happening, do this. If this isn't happening anymore, break the loop.

The JavaScript Scope:

JavaScript Code:

Google Chrome Console:

Every programming language has what is called a scope.

The scope refers to the area of code where something is defined or called upon. In JavaScript, we have the gloabl scope and the local scope.

When we define something in the global scope, it can be accessed by functions within the same scope, so for example, I have defined a variable called myVar in the global scope and set it equal to a value of 10.

What this means is that, when called in the global scope, myVar = 10. However, now check out line 4 through 7, I have defined another variable... with the same name and set it equal to 5.

This variable has been defined inside a function, which makes it a local varibale of that function, when I ask what that value is equal to whilst inside that function, the result will be 5.

Then, on line 9, I have gone back into the global scope and asked what myVar is equal to, it returns the value 10 to me.

If you are looking to define a value that can be accessed anywhere in the JavaScript file, let's say that you want a variable to store the same value everywhere and in all scopes;

To achieve this result you would use the following code; varName = value; so it is exactly the same, but you just don't say var before you define the variable.
